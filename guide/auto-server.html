            # [서버재부팅시 Tomcat 자동시작]

            서버점검 및 정전등의 이유로 서버가 재부팅될 시에 Tomcat 서비스가 자동으로 시작되도록 설정

            # 1. tomcat.service 파일 생성

            TIP

            생성 경로 및 서비스명은 동일하게 하지 않아도 된다. 모든 작업은 root 권한으로 진행한다.

            $ cd /usr/lib/systemd/system
$ vi tomcat.service

[Unit]
Description=Apache Tomcat 8.5  --  서비스 설명
After=network.target syslog.target  -- 등록되어 있는 서비스가 우선 실행되어야 tomcat 서비스가 실행

[Service]
Type=forking
User=webservice  -- tomcat 서비스를 실행할 유저
Group=webservice -- tomcat 서비스를 실행할 유저의 그룹 

ExecStart=/usr/local/tomcat7/bin/startup.sh
ExecStop=/usr/local/tomcat7/bin/shutdown.sh

[Install]
WantedBy=multi-user.target

            # 2. 데몬 재로드

            $ systemctl daemon-reload

            # 3. 시작서비스 등록

            $ systemctl enable /usr/lib/systemd/system/tomcat.service

            # 4. 테스트 ( 등록된 Tomcat은 shutdown 된 상태로 )

            $ systemctl start tomcat

            # 5. 서버재기동 후 테스트

            서버재기동이 가능한 환경에서만!!!
