    # [CentOS7] nginx 설치하기

    작성자 : 막내뤼👍

    yum명령어를 이용한 설치방법과 압축된 파일을 컴파일해 설치하는 방법을 알아보자

    • yum 명령어를 이용한 자동 설치 ( root 계정 )
    • 압축파일을 다운받아 원하는 경로에 설치 ( 사용자 계정 )

    # nginx 설치 확인

    nginx -v

    설치된 경우

    nginx version: nginx/[해당버전]

    설치가 되지 않은 경우

    -bash: nginx: command not found

    설치가 되지 않은 경우 해당 게시글을 보며 nginx설치를 따라해보자.

    # 설치

    # 1. yum 명령어를 이용한 자동설치

    # 1.1 yum 저장소에 nginx 추가

    nginx는 yum 저장소에 없기 때문에 파일을 추가 해야한다.

    cd /etc/yum.repos.d

vi nginx.repo

    [nginx]
name=nginx repo
baseurl=http://nginx.org/packages/centos/7/$basearch/
gpgcheck=0
enabled=1

    TIP

    baseurl의 경우 해당 os와 버전에 맞춰 작성하면 된다.

    제대로 yum저장소에 등록이 되었는지 확인해보자. 

    yum info nginx
    잘 되었는 경우 다음과 같이 나온다. 
    Available Packages
Name  : nginx
Arch  : x86_64
Epoch : 1
Version : 1.18.0
Release : 2.el7.ngx
(생략)

    # 1.2 nginx 설치

    yum install -y nginx
    • 설치 경로는 /etc/nginx 이다.

    # 1.3 nginx 실행

    systemctl start nginx

    # 2. wget 혹은 FTP을 이용한 수동 설치

    # 2.1 nginx 설치

    TIP

    nginx 다운로드
    zlib 다운로드

    FTP 혹은 wget을 이용하여 파일을 다운로드 받는다.

    wget https://nginx.org/download/nginx-1.18.0.tar.gz
wget https://zlib.net/zlib-1.2.11.tar.gz
    tar -xvf nginx-1.18.0.tar.gz
tar -xvf zlib-1.2.11.tar.gz
cd nginx-1.18.0

    ./configure --prefix=[설치경로] --with-zlib=../zlib-1.2.11
make
make install

    # 2.2 nginx 실행

    vi [설치경로]/conf/nginx.conf
    user  [실행할 유저];
worker_processes  1;
    cd ../sbin/
./nginx
./nginx -s stop //종료

    WARNING

    nginx로 reverse proxy 를 사용할 때, 용량이 큰 파일을 업로드하면 "413 Request Entity Too Large" 라는 메시지를 볼 수 있다. 

    vi [설치경로]/conf/nginx.conf
    http {
client_max_body_size [용량];
...
}
    기본은 1M으로 무제한으로 설정하고 싶다면 0으로 설정하면 된다.

    TIP

    # 2.2.1 systemctl 등록

    cd /usr/lib/systemd/system
vi nginx.service

    [Unit]
Description=The NGINX HTTP and reverse proxy server
After=syslog.target network.target remote-fs.target nss-lookup.target

[Service]
Type=forking
PIDFile=/run/nginx.pid
ExecStartPre=[설치경로]/sbin/nginx -t
ExecStart=[설치경로]/sbin/nginx
ExecReload=[설치경로]/sbin/nginx -s reload
ExecStop=/bin/kill -s QUIT $MAINPID
PrivateTmp=true
[Install]
WantedBy=multi-user.target

    systemctl daemon을 재시작한 후 상태를 확인해준다. 

    systemctl daemon-reload
systemctl status nginx

    # nginx 톰캣 연동

    # 3. nginx + Tomcat 연동

    # 3.1 nginx 설정

    vi  [설치경로]/conf/nginx.conf

    server {
listen 80;
server_name localhost;

#charset koi8-r;
#access_log /var/log/nginx/host.access.log main;

location / {
root /usr/share/nginx/html;
index index.html index.htm;
proxy_pass http://localhost:8090;
}
(생략)

    # 3.2 tomcat 설정

    tomcat의 server.xml에서 파일이 시작할 경로를 잡아준다. 

    Context path="" docBase="/APP/jiniworks_cms" debug="0" reloadable="false" URIEncoding="utf-8" /

    # 3.3 서버 재시작

    systemctl restart nginx
