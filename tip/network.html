# 내부망에 연결된 PC에서 외부망 연결

상황 : 내부망에 접속되는 노트북이 한대 있어서 무조건 들어가서 작업해야 하는 경우

WARNING

제약사항: 내부망에 접속해놓은 노트북의 모바일 핫스팟 기능이 가능해야 한다.
접근 제어 프로그램에 따라 핫스팟이 켜지는걸 막는 경우가 있을수 있다. 이경우 대략 난감..

# 1. 내부망에 연결된 PC의 모바일 핫스팟을 켜고 설정한다.(Mac, Windows 모두 가능. 검색할 것.)

# 2. 모바일 핫스팟이 꺼지지 않도록 설정한다.(전원 설정, 모바일 핫스팟 자체 설정 등등)

# 3. 외부망에 연결할 PC를 추가로 한대 투입한다.

# 4. 추가 투입된 PC에 wifi 연결 2개를 확보한다.

# 5. 연결중 하나를 내부망 PC의 모바일 핫스팟에 연결한다.

# 6. 나머지 하나는 포켓파이에 연결한다.


WARNING

위의 상태로는 내부망과 외부망이 아직 구별되지 않으므로 라우팅 테이블을 잡아줘야 한다.

# cmd창을 관리자 모드로 실행한다.

# 1. 기존 routing table을 모두 지워준다. 만약 실수 했을 경우 이걸 반복하면 된다.
$ route -f 

# 2. 인터페이스 목록을 확인한다. 이때 확인할 내용은 인터페이스의 번호이다.
$ route print

# 3. 외부망/내부망에 각각 연결된 네트워크의 gateway를 확인한다.
$ ipconfig /all

# 4. 기본적으로 모든 ip가 외부망 인터페이스를 타도록 설정한다.
$ route add 0.0.0.0 mask 0.0.0.0 [외부망 gateway] if [외부망 인터페이스 번호]    


# 5. 내부망 대역인 경우 내부망 인터페이스를 타도록 설정한다. 
$ route add [내부망대역] mask [내부망대역 마스크] [내부망 gateway] if [내부망 인터페이스 번호]

내부망대역과 내부망대역 마스크는 아래와 같이 설정한다.

내부망대역 마스크값
X.0.0.0 255.0.0.0
X.X.0.0 255.255.0.0
X.X.X.0 255.255.255.0
X.X.X.X 255.255.255.255

보통 대역으로 잡을테니 위의 값중 맞는 값을 찾아 넣는다.

위와 같이 설정해서 잘 안되면 route -f로 초기화후 확인후 재진행.

잘 된다면 역시 route -f 명령으로 초기화하고 4,5번의 route add 명령에 -p를 붙여서 부팅해도 적용되도록 설정한다.

이제 외부망 연결 PC에 팀뷰어를 깔고 쓴다.
